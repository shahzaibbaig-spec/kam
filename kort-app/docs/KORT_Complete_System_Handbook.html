4) Role Guides

Receptionist

Patient CreateDoctor AssignmentAdmissions

  1. Search patient by CNIC / patient number / name.
  2. If new, register patient (auto patient number generated).
  3. Assign doctor and checkup details.
  4. Create visit/admission and route patient to doctor queue.

Cannot access inventory/procurement/asset modules.

Doctor / Consultant

VisitDiagnosisPrescription

  1. Open Doctor Queue and assigned patients.
  2. Create visit note and diagnosis.
  3. Create prescription and print/download.
  • Prescription edit allowed only within 24 hours.
  • Prescription cannot be edited once dispensing starts.

Pharmacist / Medical Store Staff

Dispensing OnlyMedicine Availability Search

  1. Open Pharmacy > Prescription Lookup.
  2. Search by CNIC/patient number/prescription number to fetch prescription.
  3. Search by medicine name/code to check stock availability.
  4. Enter dispense quantities and click Done.

On Done, inventory and batches are reduced automatically and dispensing record is saved.

Cannot modify inventory master, batches, procurement, suppliers, or stock adjustments.

5) Core Module Workflows

Patient Management

  • Registration with CNIC validation and duplicate prevention
  • Auto patient number format: KORT-PAT-YYYY-000001
  • Admissions, visits, diagnoses, prescriptions, history timeline

Pharmacy Dispensing

  • FEFO batch logic
  • Expired/quarantined/damaged stock blocked
  • Status updates: pending / partially_dispensed / fully_dispensed
  • Dispensing slip print support

Inventory / Procurement / Assets / Maintenance

  • Controlled by dedicated operational/admin roles only
  • Route-level permission checks enforced

6) Audit Logging

All critical operations are logged, including:

  • Authentication events
  • Patient create/update/admission/doctor assignment
  • Diagnosis/prescription actions and print events
  • Pharmacy search/view/dispense/print/fail events
  • Inventory transactions, including pharmacy_dispensed

7) Quick Validation Checklist

  1. Receptionist can register and assign doctor, but gets 403 on inventory routes.
  2. Doctor can diagnose/prescribe, but gets 403 on inventory routes.
  3. Pharmacist can search medicine availability and dispense with Done button.
  4. Dispensing reduces stock and updates prescription dispensing status.

8) Production Recommendations

  • Rotate demo passwords immediately.
  • Enable HTTPS and production session hardening.
  • Schedule DB backups and log retention.
  • Review role matrix periodically (least privilege).
