1. Executive Summary

KORT is a role-based hospital operations system designed for burn center workflows. It links administration, organization setup, asset lifecycle, inventory lifecycle, procurement approvals, goods receiving, maintenance, calibration tracking, and audit logging in one permission-controlled application.

Core flow in one line:
Login -> role-aware dashboard -> master setup (users/departments/locations) -> register assets and items -> run issue/return/transfer/adjustment workflows -> submit requisitions -> approve -> issue purchase orders -> receive goods into inventory/assets -> monitor maintenance and calibrations -> review audit logs and settings.

2. System Architecture Snapshot

  • Backend: Laravel 13, PHP 8.3+
  • Frontend: Inertia.js + Vue 3 + Tailwind CSS
  • Auth: Session-based (Laravel Breeze)
  • Authorization: Spatie Laravel Permission + Policies
  • Audit: Spatie Activity Log
  • Data Store: MySQL/MariaDB (production), SQLite (light local)
  • Barcode/QR/Print Foundation: integrated in asset labels and print views

3. Login and Access Model

  • Users log in with email/password on /login.
  • Self-registration is intentionally disabled for safety/compliance.
  • After login, users land on /dashboard.
  • Sidebar navigation is generated by permissions; users only see modules they are allowed to access.
  • Every sensitive operation (asset, inventory, procurement, maintenance) is permission-gated and auditable.

5. Complete End-to-End App Flow

Step 1: Administrative and Organization Setup

Goal: establish the hospital structure and access control baseline.
  • Create and manage users (admin roles).
  • Assign roles and permissions.
  • Create departments (e.g., BICU, CBST, BME).
  • Create locations (wards, stores, rooms, isolation points, reserve storage).

Step 2: Asset Foundation and Lifecycle

Goal: register physical equipment and track all custody/state changes.
  1. Create asset categories.
  2. Register assets with department/location/supplier details.
  3. Generate single or bulk tags (barcode/QR), print labels.
  4. Issue assets to staff/location, return assets, transfer assets.
  5. Change condition/status with reason and full status log.
  6. Scan by tag/barcode/QR/asset-code/serial for quick lookup.

Each change writes movement history + activity audit entries.

Step 3: Inventory Foundation and Stock Workflows

Goal: maintain batch-safe medical stock with traceable movements.
  1. Create inventory categories and items (reorder/min/max, risk flags, storage flags).
  2. Receive stock into batches.
  3. Issue stock using FEFO allocation (earliest valid expiry first).
  4. Process returns with condition outcomes: usable, damaged, contaminated/quarantine, expired.
  5. Transfer stock between locations.
  6. Adjust stock (increase/decrease/recount/quarantine/release/damage/expiry).
  7. Review complete inventory ledger by item/batch/type/date/location.
  8. Scan item barcode or batch for quick retrieval.

Step 4: Procurement Lifecycle

Goal: convert demand into controlled purchasing and receiving.
  1. Manage supplier master data.
  2. Create purchase requisitions (asset/inventory lines), keep as draft or submit.
  3. Approval routing:
    • Level 1: Department Head / Manager
    • Level 2: Hospital Admin for high-value requisitions
  4. Create purchase orders from approved requisitions.
  5. Issue PO to supplier, then receive goods through GRN.
  6. Auto integration:
    • Inventory lines create/update inventory batches and transactions.
    • Asset lines create new asset records in receiving hold/state.
  7. If rejected quantity/discrepancy exists, GRN is flagged.
  8. PO moves to partial/full received, then can be closed.

Step 5: Maintenance and Calibration

Goal: keep biomedical equipment reliable and fit for clinical use.
  • Create maintenance tickets (corrective/preventive/calibration/inspection/breakdown/warranty).
  • Track ticket status: open -> assigned -> in_progress -> awaiting_parts -> completed -> closed.
  • Set fit status: fit_for_use, unfit_for_use, conditional.
  • Use calibration schedule view with due states: upcoming, due_soon, overdue.

Step 6: Security, Reporting, and Settings

  • Audit logs show who did what, when, and in which module.
  • Dashboard presents operational metrics, alerts, and role-aware quick actions.
  • Settings manage general defaults, label/barcode behavior, and notification preferences.

6. Module-by-Module Features

Administration

  • User management: create, edit, assign role, lifecycle control.
  • Role management: permission matrix updates.
  • Permission-aware navigation and policies across all modules.

Organization

  • Department master with type, code, manager, cost center, clinical flags.
  • Location hierarchy with storage type, isolation, emergency reserve, sterile flags.

Assets

  • Asset registry CRUD with supplier/department/location/financial metadata.
  • Asset categories and classification.
  • Tag generation (single/bulk) and label print/bulk print.
  • Issue, return, transfer, status-change workflows.
  • Movement trail + status logs + scan lookup.

Inventory

  • Inventory items with safety and handling indicators.
  • Batch-level stock control and expiry-sensitive processing.
  • Stock receipt/issue/return/transfer/adjustment transactions.
  • FEFO allocation and blocked expired/quarantined issue behavior.
  • Ledger and scan workflows.

Procurement

  • Supplier lifecycle management.
  • Requisition draft, submit, approve, reject, cancel.
  • Purchase order draft, issue, cancel, close.
  • Goods receipt with accepted/rejected quantity handling.
  • Cross-module integration into inventory and assets.

Maintenance and Biomedical

  • Ticket creation and status operations.
  • Engineer/supplier assignment, downtime and spare parts tracking.
  • Calibration schedule and due-state filtering.
  • Fit/unfit certification support.

Security and Settings

  • Audit log search by user/event/module/date.
  • General settings: product/organization/date/currency/timezone/thresholds.
  • Label settings: tag pattern, size, barcode/QR toggles, print layout.
  • Notification settings: low stock, maintenance, approvals, digest controls.

9. Suggested Demo Script (Fast Walkthrough)

  1. Login as Hospital Admin, show dashboard metrics and quick actions.
  2. Open Users/Roles to explain permission-based access.
  3. Open Departments and Locations to show organizational model.
  4. Switch to Biomedical Engineer: create/view an asset, generate tag, print label, run status change.
  5. Switch to Store Manager: receive stock, then issue stock to demonstrate FEFO and ledger entries.
  6. Switch to Nurse Supervisor: create and submit a requisition.
  7. Approve as Burn Center Manager (and as Hospital Admin for high-value case).
  8. Switch to Procurement Officer: create PO and issue it.
  9. Switch to Store Manager: process GRN with accepted/rejected quantities to show flagged handling.
  10. Open Maintenance schedule/tickets and demonstrate status/fit update.
  11. Finish with Auditor view on audit logs.

11. Final Notes

  • The app is built with a strong audit and permissions model suitable for hospital compliance contexts.
  • Inventory and procurement are tightly integrated; accepted goods can directly update stock or create assets.
  • All operational flows are implemented as explicit workflows with validations and transaction records.
